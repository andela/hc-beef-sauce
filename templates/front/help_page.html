FAQs

A check is a ping that is sent from your system to health checks showing that a particular process has been run to the end.
The link for the ping is placed within the source code of your application and should run within the specific time allocated (Timeout + Grace Period), otherwise health checks assumes it has gone down and will notify you of such.

An integration is a communication platform which health checks will use to communicate any information concerning you checks.
There are many integrations like slack, email, hip chat and others. We are continuously working to bring new integrations to create more options for you.

Once the nag feature is turned on it sends messages periodically until the user pings the specific check or they pause the monitoring of that check.
The time between each of the nagging messages is the the sum of the timeout and grace periods for that check.

When a check goes down there are two ways to resolve it. The first is to ping the link for that check either through your system or manually. The second way is to pause the monitoring of that check and when it goes back up, the system will receive a ping and continue monitoring as usual.

To invite team members, a user has to have a paid membership for health checks. The user can then go to account preferences and invite team members. They have to assign the team members checks that they can have access to, they can remove any of these users from the team at any time.
